# MIDIM Open-Core Boundary

This document defines the exact boundary between what is free and open-source (MIDIM OSS) and what is part of the MIDIM.net commercial platform.

## Overview

MIDIM OSS tools operate entirely locally. They perform no network calls to MIDIM.net and store nothing in any registry. They are designed for developers who want to instrument their MIDI workflows locally.

MIDIM.net provides the registry, provenance graph, rights engine, and marketplace on top of the same cryptographic primitives.

## What OSS tools will never output

To maintain the open-core boundary and avoid misleading users, MIDIM OSS tools **never** output:

- `"infringing"` / `"not infringing"` — copyright determinations are legal, not mathematical
- `"marketplace_eligible"` — eligibility requires registry-backed POIM (MIDIM.net)
- `"rights_confidence"` — confidence scoring requires corpus analysis (MIDIM.net)
- `"poimLevel"` — POIM levels 3+ require registry submission (MIDIM.net)
- Any claim that a local self-signed certificate is equivalent to a registry certificate

---

## Certificate disclaimer

Every local POIM certificate produced by `@midim/poim` includes a mandatory `disclaimer` field:

```
This is a LOCAL SELF-SIGNED certificate generated by MIDIM OSS tools.
It has NOT been submitted to or verified by any POIM registry.
It confers no marketplace eligibility, rights confidence score, or registry-backed provenance.
For registry-backed POIM, corpus similarity search, rights registry lookup,
and marketplace eligibility, visit MIDIM.net.
```

This field must not be stripped by downstream tooling.

---

## Upgrade path

Local self-signed certificates produced by `@midim/poim` are designed to be compatible with MIDIM.net registry submission. The same Ed25519 keypair and BLAKE3 asset hash used locally can be used to submit a registry-backed certificate at MIDIM.net.
